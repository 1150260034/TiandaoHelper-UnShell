# Claude Code 协作指南

## 1. 概述

本文档描述了如何与 Claude Code 进行高效的异步协作，特别是在逆向工程等复杂任务中。通过合理的工作流程，可以充分利用 Claude Code 的分析和生成能力，同时保持工作的高效性和灵活性。

## 2. 工作环境准备

### 2.1 环境配置

- **安装 Claude Code**：
  - macOS、Linux、WSL：`curl -fsSL https://claude.ai/install.sh | bash`
  - Windows PowerShell：`irm https://claude.ai/install.ps1 | iex`
  - Windows CMD：`curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd`

- **安装必要工具**：
  - ADB：Android 调试桥
  - Frida：动态分析工具
  - jadx：DEX 反编译工具
  - dex2jar：DEX 转 JAR 工具

### 2.2 终端设置

- **终端 4**：用于 Claude Code 交互
- **终端 5**：用于执行其他操作
- **多终端配置**：根据需要创建多个 Claude Code 终端，每个终端可以处理不同的任务
  - 例如：终端 4 处理脚本分析，终端 6 处理代码生成，终端 7 处理文档生成

## 3. 异步协作流程

### 3.1 启动与分配任务

1. **启动单个 Claude Code**：
   ```bash
   # 在终端 4 中执行
   claude
   ```

2. **启动多个 Claude Code 终端**：
   ```bash
   # 在终端 4 中执行
   claude
   
   # 在终端 6 中执行
   claude
   
   # 在终端 7 中执行
   claude
   ```

3. **分配任务**：
   ```bash
   # 在终端 4 中向 Claude Code 分配任务
   echo "请分析 Frida 脱壳脚本" | claude
   
   # 在终端 6 中向 Claude Code 分配任务
   echo "请创建反编译脚本" | claude
   
   # 在终端 7 中向 Claude Code 分配任务
   echo "请生成项目文档" | claude
   ```

4. **后台处理**：
   - 多个 Claude Code 终端在后台处理不同任务
   - 切换到终端 5 继续执行其他操作

### 3.2 并行操作

在 Claude Code 处理任务的同时，可以在终端 5 中执行以下操作：

1. **检查项目结构**：
   ```bash
   ls -la
   ```

2. **准备工作目录**：
   ```bash
   mkdir -p tools
   ```

3. **查看现有文件**：
   ```bash
   cat scripts/unshell_and_pull_improved.sh
   ```

4. **准备设备**：
   ```bash
   adb devices
   ```

### 3.3 检查与验证

1. **定期检查**：
   - 过一段时间后，切换到各个 Claude Code 终端检查工作状态
   - 例如：先检查终端 4 的脚本分析结果，再检查终端 6 的代码生成结果
   - 查看任务是否完成，结果是否符合要求

2. **结果验证**：
   - 检查生成的文件是否存在
   - 验证代码质量和功能完整性
   - 测试脚本是否能正常运行

3. **反馈与调整**：
   - 向各个 Claude Code 终端提供反馈
   - 要求进一步改进或调整
   - 根据任务优先级调整处理顺序

## 4. 任务类型与分配

### 4.1 适合分配给 Claude Code 的任务

- **代码分析**：分析 Frida 脚本、shell 脚本等
- **代码生成**：创建新的脚本和工具
- **代码改进**：优化现有代码，提高性能和稳定性
- **文档生成**：创建使用指南和说明文档
- **问题解决**：提供技术解决方案和建议

### 4.2 适合手动执行的任务

- **文件操作**：创建、修改和删除文件
- **工具安装**：安装和配置必要的工具
- **设备操作**：连接设备，执行 adb 命令
- **测试验证**：运行脚本，测试功能
- **环境配置**：设置系统环境和参数

## 5. 最佳实践

### 5.1 任务分配

- **明确具体**：提供清晰、具体的任务描述
- **分步骤执行**：将复杂任务分解为小步骤
- **提供上下文**：分享相关代码和项目信息
- **设定期限**：为任务设定合理的时间限制

### 5.2 结果验证

- **代码审查**：检查生成的代码是否符合要求
- **功能测试**：测试脚本是否能正常运行
- **性能评估**：评估代码的效率和可靠性
- **文档验证**：确保文档内容准确完整

### 5.3 工作协调

- **定期沟通**：保持与各个 Claude Code 终端的定期沟通
- **进度跟踪**：监控多个任务的执行进度
- **资源管理**：合理分配系统资源，避免过多终端导致性能下降
- **冲突避免**：避免同时执行可能冲突的操作
- **任务优先级**：根据任务重要性和紧急性调整处理顺序
- **终端管理**：根据任务类型和复杂度合理分配终端

## 6. 案例研究

### 6.1 逆向工程工作流程

**单终端版本**：
1. **准备阶段**：
   - 启动 Claude Code
   - 分配任务：分析 Frida 脱壳脚本
   - 并行准备设备和工具

2. **脱壳阶段**：
   - 让 Claude Code 改进脱壳脚本
   - 并行执行 adb 命令连接设备
   - 检查脱壳脚本改进结果

3. **反编译阶段**：
   - 让 Claude Code 创建反编译脚本
   - 并行运行脱壳脚本提取 DEX
   - 检查反编译脚本创建结果

4. **分析阶段**：
   - 让 Claude Code 分析反编译后的代码
   - 并行整理项目结构
   - 检查分析结果

**多终端版本**：
1. **准备阶段**：
   - 启动 3 个 Claude Code 终端
   - 终端 4：分析 Frida 脱壳脚本
   - 终端 6：创建反编译脚本
   - 终端 7：生成项目文档
   - 并行准备设备和工具

2. **脱壳阶段**：
   - 终端 4：改进脱壳脚本
   - 终端 6：优化反编译脚本
   - 终端 7：编写脱壳流程文档
   - 并行执行 adb 命令连接设备
   - 检查各终端的工作结果

3. **反编译阶段**：
   - 终端 4：分析提取的 DEX 文件
   - 终端 6：运行反编译脚本
   - 终端 7：编写反编译指南
   - 并行整理项目结构
   - 检查各终端的工作结果

4. **分析阶段**：
   - 终端 4：分析反编译后的代码结构
   - 终端 6：识别关键功能模块
   - 终端 7：生成完整的分析报告
   - 并行验证分析结果
   - 整合各终端的分析结果

### 6.2 常见问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| Claude Code 响应缓慢 | 分配更具体的任务，减少复杂性 |
| 生成的代码有错误 | 提供更详细的要求，要求错误处理 |
| 环境限制导致文件操作失败 | 手动实现文件操作，让 Claude Code 专注于分析和建议 |
| 任务执行超时 | 分解任务为更小的步骤，增加时间限制 |

## 7. 总结

通过异步协作流程，可以充分利用 Claude Code 的分析和生成能力，同时保持工作的高效性和灵活性。这种工作方式特别适合逆向工程等复杂任务，能够显著提高工作效率和质量。

关键优势：
- **提高效率**：并行处理多个任务
- **充分利用资源**：同时使用 Claude Code 和手动操作
- **灵活性**：根据需要随时切换任务
- **质量保证**：通过验证确保结果质量

通过遵循本文档中的工作流程和最佳实践，可以建立一个高效、可靠的 Claude Code 协作机制，为逆向工程等复杂任务提供有力支持。
